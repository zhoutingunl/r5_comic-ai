# 产品愿景与思考

## 1. 目标用户、痛点与用户故事
计划将小说转漫画生成器定位给以下三类用户：

**创作者与独立作者**
- *痛点*: 缺乏美术资源、图像产出周期长、难以将文本快速可视化。
- *用户故事*: 小说作者希望在每章更新后快速产出 3-4 张漫画感的宣传图。她在应用中粘贴最新章节内容，挑选角色参考图，几分钟内生成可用于社媒宣传的漫画片段。

**网文出版发行公司或短视频小说解说**
- *痛点*: 需要在短视频中快速呈现小说内容，传统制作流程效率低下，文本朗读难以带来视觉冲击。
- *用户故事*: 某网文出版公司为热门小说制作短视频解说。他们使用该应用将小说章节转换为漫画分镜，并结合文本朗读，提升视频的视觉吸引力和用户留存。

**同人创作**
- *痛点*: 小说读者希望将自己喜爱的小说内容以漫画形式呈现，但缺乏绘画技能和资源。
- *用户故事*: 一位小说爱好者希望将自己喜欢的小说片段以漫画形式分享给朋友。

## 2. 实现挑战与应对策略

**文本解析的准确性与结构化**
- *挑战*: 小说文本存在行文风格差异、对话混杂、人物指代不明确等问题。
- *应对*: 使用针对中文小说调优的 Prompt 模板，配合 `analyzeNovel` 的 Zod Schema 强制结构化；引入场景编辑器允许用户校正。

**角色一致性与图像质量**
- *挑战*: 不同面板中角色外观可能不一致，影响用户信任度。
- *应对*: 建立角色三视图库，生成时自动注入角色参考；


## 3. 模型选择与理由

**阿里云 DashScope 平台的 Qwen 系列模型**：
- 文本理解与结构化：`qwen-plus`
- 图像生成与编辑：`qwen-image-plus` / `qwen-image-edit`


选择 DashScope 的核心原因：
1. **口碑与性能**：Qwen 系列开源模型口碑良好，性能优秀，更容易宣传。
2. **多模态能力**：Qwen-image 系列在图像生成与编辑方面表现出色，适合漫画风格需求。
3. **成本**：百炼提供免费额度，适合初期验证与迭代。

## 4. 未来规划与重要性

**优化Prompt**：
- 持续优化文本解析与图像生成的 Prompt，提升输出质量与一致性。

**更新工作流**：
- 引入更多用户自定义选项，如面板布局、风格选择等，提升用户体验。
- 不一次生成图片而是先生成草稿，用户确认后再生成最终图像，防止模型能力不足导致的错误。
**实现场景一致性**：
- 通过上下文，确保同一场景中角色与背景的一致性。
**精确文本生成**：
- 现有模型难以一次生成准确图片和文本，考虑拆分为多步生成，先生成图像再添加文本。
